👋 关于我

嗨，我是 Eric，一个产品经理。

白天是产品经理，思考"为什么做"和"给谁用",
夜晚是工程师，沉迷于 Vibe Coding.

两个身份听起来矛盾，但我觉得好的软件本来就该是产品思维和工程美学的交汇。

我不会写代码（能看懂一点）。不会 Go，不会 TypeScript，不会 Vue——但这不妨碍我正在构建一个包含完整后端架构、多引擎渲染、MCP 服务的桌面应用。

怎么做到的？—— Vibe Coding。

我把产品经理的需求分析、系统设计、架构思维，和 AI 的代码生成能力结合在一起。我负责想清楚"做什么"和"为什么这样做"，AI 负责把它变成可运行的代码。每一行代码都经过我的审视——不是逐字审查语法，而是从产品和架构的视角确认它是否合理、清晰、可维护、可扩展，最重要的：是否是最佳实践。

🏠 关于这个站

这个站点就是使用新版的 Gridea Pro 搭建的。

Gridea 是一个很棒的开源静态博客客户端，GitHub 上 10k+ stars，帮助了 28000 多位用户搭建自己的博客。我很喜欢它的理念——让非技术用户也能轻松拥有自己的博客。

但原版的技术栈（Electron + Vue 2 + JavaScript）已经到了瓶颈，项目也逐渐停止了维护。于是我决定基于 Gridea 的产品理念，用现代技术栈进行全面升级重构：Go + Wails + Vue 3 + TypeScript。不只是换个皮肤，而是从架构层面重新思考——四层分层架构、泛型 JSON 持久化、多引擎渲染（EJS / Jinja2 / Go Template）、完整的 MCP 服务支持。

🧠 我在想什么

最近脑子里装着这些东西：

  • AI Agent 的未来形态 — 从 Chat 到 Agent，从对话到协作。我相信 Agentic Workflow 会彻底改变软件的交互范式，不是"更好的聊天机器人"，而是"能自主完成任务的数字同事"
  • MCP 协议的生态可能性 — 我已经为 Gridea Pro 实现了完整的 MCP 服务（28 个 Tools、3 个 Resources、5 个 Prompts），让 AI 可以直接操作博客。这篇关于页面？说不定就是 AI 帮我发布的 😏
  • Vibe Coding 的边界在哪 — 一个不会写代码的人，到底能用 AI 把产品做到什么程度？我正在用 Gridea Pro 亲自验证这件事
  • 系统设计的美感 — 不需要亲手写代码，也可以对架构有追求。一个好的系统应该像一首诗，每一层都有它的韵律

🧩 一些碎片

  • 🦉 很多文章写于凌晨 2-3 点，夜深人静的时候思考和写作效率最高
  • 🌏 会关注一些海外大神如 Dan Koe、Karpathy 等创作者关于 AI 的前沿思想
  • 🤖 Vibe Coding 重度实践者，相信产品思维 + AI 能力是未来个人开发的最佳组合
  • 🎯 产品经理的职业病：看到任何 App 都会下意识分析它的信息架构和交互逻辑
  • 📐 虽然不手写代码，但对架构有洁癖——分层要清晰，命名要准确，职责要单一
  • 🛠️ 工具控，但不盲从——好的工具应该让你忘记工具本身的存在
