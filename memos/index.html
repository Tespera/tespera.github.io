少 多

用 GUI 的极致体验降低静态博客的使用门槛，同时通过开放的主题/插件生态保证可扩展性，让"不会写代码但追求高质量博客"的用户群体拥有一个真正好用的工具。

2026-02-25

竞争定位： Hexo 和 Hugo 都没有成熟的桌面 GUI——这是 Gridea Pro 最大的差异化机会。不要在 CLI 维度与它们正面竞争，而是做"静态博客领域的 Notion"，用 GUI 体验 + 开放生态 + AI 集成来服务不愿折腾命令行的用户群体。

2026-02-25

内置 SEO 与分析

静态博客普遍缺乏的能力：

内置 SEO 检查清单（标题优化、meta description、Open Graph、结构化数据）
集成 Umami/Plausible 等隐私友好的开源分析工具
Sitemap 和 RSS 自动生成和优化
社交分享预览（发布前预览在 Twitter/微信等平台的分享卡片效果）

2026-02-25

开放的主题/插件生态系统

这是与 Hexo/Hugo 竞争的关键：

  • 设计规范化的主题 API 和模板规范，降低主题开发门槛
  • 建设主题市场（内置浏览、预览、一键安装），参考 VS Code 扩展市场的体验
  • 支持插件系统（评论、搜索、统计、SEO 等功能可通过插件添加）
  • 提供主题/插件开发 SDK 和文档，吸引社区贡献
  • 可考虑兼容 Hugo/Hexo 主题格式，直接继承其庞大的主题库
2026-02-25

编辑器是 Gridea 作为"写作客户端"的核心体验，但当前编辑器被评价为"对小白还差点意思"：

  • 采用 Milkdown、Tiptap 或 BlockNote 等现代编辑器框架，实现 Notion 级别的块编辑体验
  • 支持所见即所得（WYSIWYG）与 Markdown 源码双模式无缝切换
  • 内置图片拖拽上传 + 多图床支持（GitHub、S3、Cloudflare R2、SM.MS 等）
  • 支持实时预览（编辑器左侧编辑，右侧实时预览最终页面效果）
  • AI 辅助写作能力（语法检查、摘要生成、SEO 标题建议等）
2026-02-25
